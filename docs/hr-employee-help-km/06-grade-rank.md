# ៦) គ្រប់គ្រងថ្នាក់ និងឋានន្តរស័ក្តិ

## 6.1 គោលបំណង

ផ្នែកនេះសម្រាប់គណនា "អ្នកគ្រប់លក្ខខណ្ឌ", បង្កើតសំណើ, អនុម័តសំណើ, និងតាមដានរបាយការណ៍ឡើងថ្នាក់/ឋានន្តរស័ក្តិ។

## 6.2 គោលការណ៍គណនា (Standard)

- ករណីប្រចាំឆ្នាំ: គ្រប់រយៈពេល **2 ឆ្នាំឡើងទៅ**
- គិតលក្ខខណ្ឌតាម **Cutoff date** (ប្រើថ្ងៃកំណត់របស់ស្ថាប័ន)
- ករណីពិសេស (សញ្ញាបត្រ/កិត្តិយសមុននិវត្តន៍) ត្រូវមានឯកសារគាំទ្រ

## 6.3 អត្ថន័យ Tab នីមួយ

1. ផ្ទាំងសង្ខេប: មើល KPI និងស្ថានភាពទូទៅ
2. បញ្ជីគ្រប់លក្ខខណ្ឌ: តារាងបេក្ខភាពដែលស្នើបាន
3. ព័ត៌មានមន្រ្តី: ពិនិត្យ profile មុនបង្កើតសំណើ
4. បង្កើតសំណើ: បញ្ចូលព័ត៌មានសំណើសម្រាប់មន្រ្តី
5. ប្រវត្តិដំឡើង: មើលអតីតការដំឡើង
6. សំណើទាំងអស់: តាមដាន Draft/Pending/Approved/Rejected
7. ការអនុម័ត: អនុម័តជាបុគ្គល ឬជាក្រុម
8. របាយការណ៍: Export/Print
9. ការជូនដំណឹង: រំលឹកករណីដល់ពេល/លើសពេល

## 6.4 លំហូរដែលណែនាំ

1. កំណត់ឆ្នាំ និង cutoff date
2. ចុច Recalculate
3. ពិនិត្យបញ្ជីគ្រប់លក្ខខណ្ឌ
4. ជ្រើសមន្រ្តី បង្កើតសំណើ
5. បញ្ចូលឯកសារយោង និងចំណាំ
6. Submit ទៅអនុម័ត
7. Approver ពិនិត្យ និងសម្រេច
8. Approved -> update record និងប្រវត្តិ

## 6.5 ស្ថានភាពសំណើ (ឧទាហរណ៍)

- Draft: កំពុងរៀបចំ មិនទាន់ដាក់ស្នើ
- Pending: ដាក់ស្នើរួច កំពុងរង់ចាំអនុម័ត
- Approved: អនុម័តរួច
- Rejected: មិនអនុម័ត
- Returned: ត្រឡប់ឱ្យកែ

## 6.6 ចំណុចពិនិត្យសំខាន់ពេលអនុម័ត

1. ថ្នាក់បច្ចុប្បន្នត្រឹមត្រូវ
2. ថ្នាក់ត្រូវឡើងត្រឹមត្រូវ (មិនរំលងខ្សែ)
3. មានលិខិតយោង
4. មិនស្ទួនសំណើឆ្នាំដូចគ្នា

## 6.7 កំហុសជាញឹកញាប់

- ប៊ូតុង/Tab ស្ទួន: ប្រើលំហូរមូលដ្ឋានតែ 1 ជួរ
- អក្សរខ្មែរ​មិនបង្ហាញ: ពិនិត្យ language key និង encoding
- ចំនួន Pending/Approved មិនដូចគ្នា: ពិនិត្យ filter និង status definition

## 6.8 Checklist ប្រចាំឆ្នាំ

1. កំណត់ cutoff date ត្រឹមត្រូវ
2. Recalculate ម្តងមុនស្នើ
3. ពិនិត្យអ្នក over-due
4. ពិនិត្យឯកសារខ្វះ
5. អនុម័ត និងបិទវដ្តការងារ
