# សៀវភៅជំនួយការប្រើប្រាស់ប្រព័ន្ធគ្រប់គ្រងឱសថ

> ចំណាំ: Help នេះត្រូវបានបំបែកជា module នីមួយៗរួចហើយ នៅក្នុង folder `docs/pharmacy-help-km/` ដើម្បីឲ្យងាយស្រួលប្រើប្រាស់ជាងមុន។ ឯកសារនេះនៅតែរក្សាទុកជាឯកសារមេ សម្រាប់មើលជាសរុប។

## 1. គោលបំណង

ឯកសារនេះត្រូវបានរៀបចំឡើងសម្រាប់ជួយអ្នកប្រើប្រាស់ប្រព័ន្ធគ្រប់គ្រងឱសថ ក្នុងការប្រើមុខងារសំខាន់ៗប្រចាំថ្ងៃ ដូចជា ការបញ្ចូលឱសថ ការចែកចាយឱសថ ការផ្តល់ឱសថដល់អ្នកជំងឺ ការកែសម្រួលស្តុក និងការធ្វើរបាយការណ៍។

ប្រព័ន្ធនេះមានគោលបំណងដើម្បី:

- គ្រប់គ្រងបញ្ជីឱសថឲ្យមានស្តង់ដារ
- តាមដានស្តុកឱសថតាមមុខឱសថ និង batch
- កត់ត្រាការចែកចាយ និងការផ្តល់ឱសថ
- គាំទ្រការរៀបចំរបាយការណ៍ប្រចាំខែ ឬប្រចាំកាលបរិច្ឆេទ
- បន្ថយកំហុសក្នុងការគណនាស្តុក និងការរាយការណ៍

## 2. មុនពេលចាប់ផ្តើម

មុនប្រើប្រាស់ប្រព័ន្ធ សូមពិនិត្យថា:

- អ្នកមានគណនីចូលប្រើប្រាស់រួចហើយ
- អ្នកបាន Login ចូលប្រព័ន្ធដោយជោគជ័យ
- អ្នកមានសិទ្ធិចូលប្រើ module Pharmaceutical
- អ្នកបានជ្រើស Facility ឬផ្នែកសុខាភិបាលត្រឹមត្រូវ តាមតួនាទីរបស់អ្នក

ចំណាំ:

- អ្នកប្រើប្រភេទ Admin ឬអ្នកគ្រប់គ្រងកម្រិតខ្ពស់ អាចឃើញមុខងារ Users
- អ្នកប្រើទូទៅភាគច្រើននឹងប្រើ Medicines, Distributions, Dispensing, Stock, Adjustments និង Reports

## 3. ទិដ្ឋភាពទូទៅនៃមុខងារ

ក្នុង module Pharmaceutical មានផ្នែកសំខាន់ៗដូចខាងក្រោម:

### Dashboard

ប្រើសម្រាប់មើលស្ថានភាពទូទៅនៃប្រព័ន្ធឱសថ ដូចជា ចំនួនមុខឱសថ ស្តុកសរុប និងព័ត៌មានសង្ខេបផ្សេងៗ។

### Medicines

ប្រើសម្រាប់បញ្ចូល និងគ្រប់គ្រងព័ត៌មានឱសថនីមួយៗ។

### Categories

ប្រើសម្រាប់បែងចែកប្រភេទឱសថ ដើម្បីងាយស្រួលស្វែងរក និងធ្វើរបាយការណ៍។

### Distributions

ប្រើសម្រាប់កត់ត្រាការផ្ទេរ ឬចែកចាយឱសថពី facility មួយទៅ facility មួយទៀត។

### Stock

ប្រើសម្រាប់ពិនិត្យស្តុកឱសថបច្ចុប្បន្ន។

### Adjustments

ប្រើសម្រាប់កែសម្រួលស្តុក នៅពេលមានខូចខាត ផុតកំណត់ បាត់បង់ ឬត្រូវកែតម្រូវចំនួនស្តុកជាក់ស្តែង។

### Dispensing

ប្រើសម្រាប់កត់ត្រាការផ្តល់ឱសថដល់អ្នកជំងឺ។

### Reports

ប្រើសម្រាប់បង្កើត និងគ្រប់គ្រងរបាយការណ៍ឱសថតាម facility និងតាមរយៈពេល។

### Summary Reports

មានរបាយការណ៍សង្ខេប 2 ផ្នែកសំខាន់:

- របាយការណ៍ប្រើប្រាស់ឱសថ
- របាយការណ៍ស្តុកដើម-ចុងគ្រា

### Users

ប្រើសម្រាប់កំណត់អ្នកប្រើ និងសិទ្ធិប្រើប្រាស់ module Pharmaceutical។ ផ្នែកនេះភាគច្រើនសម្រាប់ Admin ឬអ្នកគ្រប់គ្រង។

## 4. របៀបប្រើ Dashboard

Dashboard ជាទំព័រដំបូងសម្រាប់តាមដានទិន្នន័យសំខាន់ៗ។

អ្នកអាចប្រើ Dashboard ដើម្បី:

- មើលស្ថានភាពស្តុកជារួម
- ពិនិត្យចំនួនមុខឱសថដែលបានបញ្ចូល
- តាមដានទិន្នន័យសង្ខេបសម្រាប់ការសម្រេចចិត្តលឿន

ណែនាំ:

- ប្រសិនបើអ្នកចង់មើលស្តុកលម្អិត សូមចូលទៅផ្នែក Stock
- ប្រសិនបើទិន្នន័យលើ Dashboard មិនទាន់ត្រឹមត្រូវ សូមពិនិត្យថាបានកត់ត្រា Distribution, Dispensing និង Adjustment គ្រប់គ្រាន់ឬនៅ

## 5. របៀបបញ្ចូលឱសថថ្មី

ផ្នែក Medicines ប្រើសម្រាប់បង្កើត master data របស់ឱសថ។ មុននឹងចែកចាយ ឬផ្តល់ឱសថ អ្នកត្រូវមានឱសថក្នុងបញ្ជីនេះជាមុនសិន។

### ព័ត៌មានត្រូវបញ្ចូល

- Code: លេខកូដឱសថ
- Name: ឈ្មោះឱសថ
- Name (KH): ឈ្មោះឱសថជាភាសាខ្មែរ
- Category: ប្រភេទឱសថ
- Dosage form: ទម្រង់ឱសថ ដូចជា tablet, capsule, syrup
- Strength: កម្រិតឱសថ ដូចជា 500mg, 250mg/5ml
- Unit: ឯកតា ដូចជា tablet, bottle, box
- Manufacturer: ក្រុមហ៊ុនផលិត
- Unit price: តម្លៃក្នុងមួយឯកតា

### ជំហានបញ្ចូល

1. ចូលទៅកាន់ផ្ទាំង Medicines
2. ចុចប៊ូតុងបង្កើតឱសថថ្មី
3. បញ្ចូលព័ត៌មានតាម fields ដែលត្រូវការ
4. ពិនិត្យ Code, Name និង Unit ឲ្យត្រឹមត្រូវ
5. ចុច Save

### ចំណាំសំខាន់

- Code មិនគួរដូចគ្នាជាមួយឱសថផ្សេងទៀត
- Unit គួរបញ្ចូលឲ្យស្របនឹងការប្រើប្រាស់ជាក់ស្តែង
- បើឱសថមានឈ្មោះជាខ្មែរ សូមបញ្ចូលក្នុង Name (KH) ដើម្បីងាយសម្រាប់អ្នកប្រើក្នុងស្រុក

### បញ្ហាដែលជួបញឹកញាប់

- មិនអាច Save បាន ព្រោះមិនបានបញ្ចូល Code ឬ Name
- Category មិនទាន់មានក្នុងប្រព័ន្ធ
- Unit បញ្ចូលមិនត្រឹមត្រូវ ធ្វើឲ្យការរាយការណ៍លំបាក

## 6. របៀបគ្រប់គ្រង Categories

Categories ជួយឲ្យការរៀបចំឱសថមានស្តង់ដារ និងងាយស្រួលក្នុងការស្វែងរក។

### ពេលណាគួរប្រើ

- ពេលបន្ថែមប្រភេទឱសថថ្មី
- ពេលចង់រៀបចំបញ្ជីឱសថតាមក្រុម
- ពេលត្រូវធ្វើរបាយការណ៍តាមប្រភេទឱសថ

### ជំហានបន្ថែម Category

1. ចូលទៅកាន់ផ្ទាំង Categories
2. បញ្ចូលឈ្មោះ category ថ្មី
3. រក្សាទុក

### ការប្រុងប្រយ័ត្ន

- កុំបង្កើត category ដូចគ្នាច្រើនដង
- ប្រើឈ្មោះដែលងាយយល់ និងស្តង់ដារ

Distribution គឺការផ្ទេរឱសថពី facility មួយទៅ facility មួយទៀត។

### ជំហានធ្វើ

6. ជ្រើសមុខឱសថក្នុងតារាង Medicine items
7. បញ្ចូល Quantity, Batch no, Expiry date និង Unit price
8. ប្រសិនបើមានឱសថច្រើនមុខ សូមចុច Add item
9. ពិនិត្យព័ត៌មានទាំងអស់ម្តងទៀត
10. ចុច Save

### ពេលណាគួរបញ្ចូល Batch no និង Expiry date

- ពេលឱសថមានការតាមដានតាម batch
- ពេលត្រូវគ្រប់គ្រងថ្ងៃផុតកំណត់ឲ្យបានត្រឹមត្រូវ
- ពេលចង់ដឹងថាស្តុកមកពីការចែកចាយណាមួយ

### កំហុសដែលជួបញឹកញាប់

- ជ្រើស From facility និង To facility ដូចគ្នា
- មិនបញ្ចូល Quantity
- បញ្ចូល Batch no ខុសពីឯកសារដើម
- មិនពិនិត្យ Expiry date មុន Save

## 8. របៀបពិនិត្យ Stock

ផ្នែក Stock ប្រើសម្រាប់មើលស្តុកបច្ចុប្បន្នរបស់ឱសថ។

អ្នកអាចប្រើផ្នែកនេះដើម្បី:

- ពិនិត្យថាឱសថណាមាននៅសល់ប៉ុន្មាន
- ប្រៀបធៀបស្តុកមុនពេលធ្វើ Dispensing ឬ Distribution
- ពិនិត្យមើលស្ថានភាពស្តុកមុនធ្វើរបាយការណ៍
- បោះពុម្ពស្តុក ប្រសិនបើត្រូវការ

ណែនាំការប្រើប្រាស់:

- សូមពិនិត្យស្តុកជាមុន មុនធ្វើការផ្តល់ឱសថដល់អ្នកជំងឺ
- ប្រសិនបើស្តុកមិនត្រឹមត្រូវ សូមពិនិត្យ Distribution, Dispensing និង Adjustments ចុងក្រោយ

## 9. របៀបកែសម្រួលស្តុក

Adjustments ប្រើនៅពេលចំនួនស្តុកជាក់ស្តែង មិនត្រូវនឹងស្តុកក្នុងប្រព័ន្ធ។

ប្រភេទ adjustment មាន:

- ខូចខាត: ដកចេញពីស្តុក
- ផុតកំណត់: ដកចេញពីស្តុក
- បាត់បង់: ដកចេញពីស្តុក
- កែតម្រូវ: បូកចូលស្តុក

### ព័ត៌មានត្រូវបញ្ចូល

- Facility
- Medicine
- Type
- Qty
- Batch
- Expiry
- Date
- Reason

### ជំហានធ្វើ

1. ចូលទៅកាន់ផ្ទាំង Adjustments

### តើ Closing stock គណនាយ៉ាងដូចម្តេច?

Closing stock = Opening + Received - Dispensed - Damaged - Expired + Adjust

### តើពេលណាគួរប្រើ Adjustment type កែតម្រូវ?

ប្រើនៅពេលរាប់ស្តុកជាក់ស្តែង ហើយឃើញចំនួនខុសពីក្នុងប្រព័ន្ធ ដែលត្រូវបូកចូលឬកែសម្រួលតាមការត្រួតពិនិត្យ។

## 18. ទំនាក់ទំនងសម្រាប់ជំនួយបន្ថែម

ប្រសិនបើអ្នកជួបបញ្ហាដែលមិនអាចដោះស្រាយបាន សូមទាក់ទង:

- អ្នកគ្រប់គ្រងប្រព័ន្ធ
- ក្រុម IT Support
- អ្នកទទួលខុសត្រូវ module Pharmaceutical

ពេលស្នើជំនួយ សូមផ្តល់ព័ត៌មានដូចជា:

- ឈ្មោះមុខងារដែលមានបញ្ហា
- ជំហានដែលអ្នកបានធ្វើ
- សារកំហុសដែលបានឃើញ
- កាលបរិច្ឆេទ និងពេលវេលាកើតបញ្ហា

## 19. សេចក្តីសន្និដ្ឋាន

ការប្រើប្រាស់ប្រព័ន្ធគ្រប់គ្រងឱសថឲ្យមានប្រសិទ្ធភាព ត្រូវការការបញ្ចូលទិន្នន័យឲ្យត្រឹមត្រូវ ទាន់ពេលវេលា និងអនុវត្តតាម flow ការងារប្រចាំថ្ងៃ។ ប្រសិនបើអ្នកប្រើ Medicines, Distribution, Dispensing, Adjustment និង Reports ឲ្យបានត្រឹមត្រូវ នោះស្តុក និងរបាយការណ៍នឹងមានភាពជឿជាក់ និងអាចប្រើសម្រាប់ការសម្រេចចិត្តបានល្អប្រសើរ។
