# ការគ្រប់គ្រងលិខិតចូល (Incoming) — មគ្គុទេសក៍លម្អិត

## ១) លិខិតចូលគឺអ្វី?

លិខិតចូលគឺឯកសារដែលអង្គភាពរបស់យើង **ទទួលចូល** ពីស្ថាប័នក្រៅ ឬអង្គភាពផ្សេងៗ ហើយត្រូវឆ្លងដំណើរការ៖

1. ចុះបញ្ជី
2. បែងចែកទៅអ្នក/អង្គភាពពាក់ព័ន្ធ
3. ប្រមូលមតិយោបល់
4. ពិនិត្យថ្នាក់ដឹកនាំ
5. សម្រេចចុងក្រោយ

---

## ២) វាលទិន្នន័យចាំបាច់ និងអត្ថន័យ

### វាលចាំបាច់

- **លេខចុះបញ្ជី**: លេខក្នុងប្រព័ន្ធ សម្រាប់យោងខាងក្នុង
- **លេខលិខិតដើម**: លេខលិខិតពីអង្គភាពផ្ញើ
- **ថ្ងៃទទួលលិខិត**: ថ្ងៃដែលការិយាល័យយើងទទួលជាក់ស្តែង
- **ថ្ងៃលិខិត**: ថ្ងៃនៅលើលិខិតដើម
- **ប្រធានបទ**: ចំណងជើងខ្លីបង្ហាញរឿងលិខិត
- **អង្គភាពចេញលិខិត**: ប្រភពលិខិត

### វាលណែនាំ

- **ខ្លឹមសារសង្ខេប**: សរសេរសង្ខេប 1-3 បន្ទាត់ ឱ្យច្បាស់
- **កំណត់សម្គាល់**: ព័ត៌មានបន្ថែមដែលជួយសម្រេច
- **ឯកសារភ្ជាប់**: PDF/DOC/រូបភាព

> ណែនាំ៖ បើលិខិតមានសន្លឹកច្រើន សូមភ្ជាប់ជា PDF មួយឯកសារ ងាយពិនិត្យជាង។

---

## ៣) ជំហានបង្កើតលិខិតចូល

1. ចូលម៉ឺនុយ `លិខិតចូល`
2. ចុច `បន្ថែមលិខិតចូល`
3. បំពេញវាលចាំបាច់ (`*`)
4. ភ្ជាប់ឯកសារ
5. ចុច `រក្សាទុក`

លទ្ធផលក្រោយ Save៖

- ស្ថានភាពទៅជា `Pending`
- លិខិតបង្កើតបាន ID ក្នុងប្រព័ន្ធ
- អាចចូលផ្ទាំង Detail ដើម្បីចាត់តាំងបន្ត

---

## ៤) ប្រគល់ភារកិច្ច / ចាត់តាំងទៅអង្គភាពពាក់ព័ន្ធ

នៅផ្ទាំង Detail នៃលិខិតចូល៖

1. ជ្រើសអ្នក/អង្គភាពពាក់ព័ន្ធ
2. បញ្ចូលសេចក្តីណែនាំឲ្យច្បាស់ (ត្រូវឆ្លើយអ្វី? ក្នុងពេលប៉ុន្មានថ្ងៃ?)
3. កំណត់អាទិភាព (ធម្មតា/បន្ទាន់)
4. រក្សាទុក

### ករណីបញ្ជូនទៅអង្គភាពច្រើន

- អាចបញ្ជូនទៅច្រើនអង្គភាពក្នុងលិខិតតែមួយ
- អង្គភាពនីមួយៗអាចឆ្លើយមតិយោបល់ខ្លួនឯង
- Action history នឹងកត់ត្រាទាំងអស់

---

## ៥) លំហូរពិនិត្យ និងសម្រេច (Incoming)

### ជំហានទី១: អង្គភាពជំនាញចំណារ
- បញ្ចូលមតិយោបល់វិជ្ជាជីវៈ
- បញ្ជាក់ច្បាស់ថា ស្នើអនុម័ត/ស្នើកែប្រែ/មិនគាំទ្រ

### ជំហានទី២: អនុប្រធានមន្ទីរពិនិត្យ
- ពិនិត្យមតិយោបល់អង្គភាពក្រោមឱវាទ
- ផ្តល់គំនិតជូនប្រធានមន្ទីរ

### ជំហានទី៣: ប្រធានមន្ទីរសម្រេច
- `Approve` បើគ្រប់លក្ខខណ្ឌ
- `Reject` បើមិនត្រឹមត្រូវ (ត្រូវមានមូលហេតុ)
- `Return for revision` បើត្រូវកែតម្រូវ

---

## ៦) ពេលណាចាត់ទុកថាបញ្ចប់?

លិខិតចូលចាត់ទុកថា **Completed** នៅពេល៖

- ប្រធានមន្ទីរបានសម្រេចចុងក្រោយ (អនុម័ត ឬ បដិសេធ)
- មិនមាន action បន្តក្នុង workflow

---

## ៧) គន្លឹះធ្វើឲ្យប្រើប្រាស់រហ័ស

- ប្រើប្រធានបទខ្លី ប៉ុន្តែច្បាស់ (កុំប្រើពាក្យទូទៅពេក)
- ឯកសារភ្ជាប់ឲ្យគ្រប់មុនចាត់តាំង
- ចំណារត្រូវបញ្ជាក់ “ស្ថានភាព + មូលហេតុ + សំណើ”
- បើមិនច្បាស់ សូម `Return for revision` ជំនួស `Reject` បន្ទាន់

---

## ៨) កំហុសជួបញឹកញាប់ និងដំណោះស្រាយ

### បញ្ហា: Save មិនបាន
- ពិនិត្យវាល `*`
- ពិនិត្យ format ថ្ងៃខែឆ្នាំ
- ពិនិត្យទំហំឯកសារភ្ជាប់

### បញ្ហា: មិនឃើញប៊ូតុងអនុម័ត
- គណនីមិនមែនជាអ្នកទទួលជំហានបច្ចុប្បន្ន
- មិនមាន Org Role/Permission ត្រឹមត្រូវ

### បញ្ហា: ចំណារមិនចេញក្នុងរបាយការណ៍បោះពុម្ព
- មិនទាន់មានមតិយោបល់តាមជំហាន
- មតិយោបល់កត់ត្រានៅ action ខុសប្រភេទ

---

## ៩) Checklist ពេលបិទលិខិតចូល

- [ ] ព័ត៌មានមូលដ្ឋានគ្រប់
- [ ] ឯកសារភ្ជាប់បើកមើលបាន
- [ ] ចំណារពីអង្គភាពពាក់ព័ន្ធមានគ្រប់
- [ ] អនុប្រធានបានពិនិត្យ
- [ ] ប្រធានបានសម្រេច
- [ ] ស្ថានភាពលិខិតទៅជា Completed/Closed
