# ៤) គ្រប់គ្រងអាជីពការងារ (Career Management)

## 4.1 គោលបំណង

ផ្នែកនេះប្រើសម្រាប់គ្រប់គ្រង "ព័ត៌មានការងារបច្ចុប្បន្ន" និង "ប្រវត្តិការងារ" ឱ្យមានទំនាក់ទំនងគ្នាត្រឹមត្រូវ។

## 4.2 ទិន្នន័យដែលគួរគ្រប់គ្រងនៅទីនេះ

- អង្គភាពបច្ចុប្បន្ន
- តួនាទីបច្ចុប្បន្ន
- ស្ថានភាពការងារ (Active, Leave without pay, Transfer out, Retirement, ...)
- ថ្ងៃមានប្រសិទ្ធភាព
- លេខលិខិត និងថ្ងៃលិខិត
- មូលហេតុ/ចំណាំ

## 4.3 គោលការណ៍ស្តង់ដារ

1. ការផ្លាស់ប្តូរស្ថានភាពការងារសំខាន់ៗ ត្រូវកត់ត្រា history ជានិច្ច
2. កុំកែ field ដែល policy កំណត់ឱ្យកែតាមម៉ូឌុល history ដោយផ្ទាល់
3. ការប្តូរអង្គភាព ឬតួនាទី ត្រូវតាមលំហូរដែលមាន workflow

## 4.4 ជំហានប្រើប្រាស់

1. ជ្រើសបុគ្គលិក
2. បើកផ្ទាំងព័ត៌មានការងារ
3. បញ្ចូលការផ្លាស់ប្តូរ (Status/Role/Org Unit)
4. បំពេញថ្ងៃមានប្រសិទ្ធភាព និងឯកសារយោង
5. Save
6. ពិនិត្យថាបានបង្កើតកំណត់ត្រាប្រវត្តិថ្មី

## 4.5 កំហុសជាញឹកញាប់

- កែ "ស្ថានភាព" នៅកន្លែងមិនត្រឹមត្រូវ -> ប្រវត្តិមិនគ្រប់
- ថ្ងៃមានប្រសិទ្ធភាពខុស -> report មិនត្រឹមត្រូវ
- គ្មានលិខិតយោង -> audit មិនគ្រប់គ្រាន់

## 4.6 Checklist

1. មានថ្ងៃមានប្រសិទ្ធភាព
2. មានប្រភេទស្ថានភាពត្រឹមត្រូវ
3. មានលេខលិខិត/ថ្ងៃលិខិត (បើតម្រូវ)
4. ប្រវត្តិថ្មីបង្កើតរួច
