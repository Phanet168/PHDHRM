# សៀវភៅណែនាំពេញលេញ — ម៉ូឌុលគ្រប់គ្រងលិខិតរដ្ឋបាល

> កំណែឯកសារ: 2.0  
> ថ្ងៃកែប្រែ: 16/04/2026

ឯកសារនេះជាគន្លឹះសរុបសម្រាប់ HR/Admin និងអ្នកប្រើប្រាស់ពាក់ព័ន្ធនឹងលិខិតចូល/លិខិតចេញ។

---

## ១) គោលបំណងប្រព័ន្ធ

ម៉ូឌុលនេះធ្វើ 5 រឿងសំខាន់៖

1. ចុះបញ្ជីលិខិតឱ្យមានស្តង់ដារ
2. បែងចែកការងារទៅអង្គភាព/អ្នកពាក់ព័ន្ធ
3. ប្រមូលមតិយោបល់តាមជំហាន
4. សម្រេចចុងក្រោយដោយថ្នាក់ដឹកនាំ
5. បោះពុម្ព និងរក្សាទុកស្នាមប្រតិបត្តិការផ្លូវការ

---

## ២) លំហូរការងារសង្ខេប

### ២.១ លិខិតចូល

`ចុះបញ្ជី -> ចាត់តាំង -> មតិអង្គភាព -> ពិនិត្យអនុប្រធាន -> សម្រេចប្រធាន -> បញ្ចប់`

### ២.២ លិខិតចេញ

`បង្កើត -> កំណត់ To/CC -> ផ្ញើ -> អ្នកទទួលបានទទួល -> បញ្ចប់`

---

## ៣) តួនាទី និងការងារ

| តួនាទី | ការងារសំខាន់ |
|---|---|
| អ្នកទទួលលិខិត | បង្កើត/ចុះបញ្ជីលិខិត |
| អ្នកគ្រប់គ្រងលិខិត | ចាត់តាំង/បែងចែក/តាមដាន |
| អង្គភាពពាក់ព័ន្ធ | ផ្តល់មតិយោបល់ |
| អនុប្រធានមន្ទីរ | ពិនិត្យ និងផ្តល់គំនិត |
| ប្រធានមន្ទីរ | សម្រេចចុងក្រោយ |
| Super Admin | កំណត់សិទ្ធិ/Workflow/Org scope |

---

## ៤) វិធីប្រើលិខិតចូល (ជាក់លាក់)

1. ចូល `លិខិតចូល`
2. ចុច `បន្ថែមលិខិតចូល`
3. បំពេញ៖ លេខលិខិត, ថ្ងៃលិខិត, ថ្ងៃទទួល, អង្គភាពចេញ, ប្រធានបទ
4. ភ្ជាប់ឯកសារ
5. Save
6. ចាត់តាំងអង្គភាពពាក់ព័ន្ធ/អ្នកពាក់ព័ន្ធ
7. ប្រមូលមតិយោបល់
8. អនុប្រធានពិនិត្យ
9. ប្រធានសម្រេច

លទ្ធផលចុងក្រោយ៖ `Completed` (Approved/Rejected)

---

## ៥) វិធីប្រើលិខិតចេញ (ជាក់លាក់)

1. ចូល `លិខិតចេញ`
2. ចុច `បន្ថែមលិខិតចេញ`
3. បំពេញព័ត៌មានលិខិត
4. ជ្រើស To/CC (អង្គភាព និងបុគ្គល)
5. ភ្ជាប់ឯកសារ
6. Save (Draft)
7. ពិនិត្យ detail ហើយចុច `ផ្ញើ`
8. តាមដានការទទួលបាន

លទ្ធផលចុងក្រោយ៖ `Completed` នៅពេលអ្នកទទួលពាក់ព័ន្ធទទួលបាន

---

## ៦) ចំណារ/មតិយោបល់ (ប្រើជាភស្តុតាង)

សូមសរសេរទម្រង់ 3 ចំណុច៖

1. បានពិនិត្យអ្វី
2. មានបញ្ហាអ្វី
3. ស្នើសុំសម្រេចអ្វី

មតិយោបល់ទាំងនេះនឹងចេញក្នុង Print report ប្រអប់មតិយោបល់ផ្លូវការ។

---

## ៧) Print & Attachment

### Print
- ចុច `Print report`
- Browser print settings: A4, Portrait, Scale 100%

### Attachment
- View/Download ពី detail
- ប្រភេទខ្លះ browser នឹង preview បាន
- ប្រភេទខ្លះ browser នឹងបើក Download អាស្រ័យ policy

---

## ៨) កំហុសញឹកញាប់

- `419 Page Expired`: session/token ផុតកំណត់
- `500`: DB/route/controller ខូច
- `404`: link notification ចាស់ ឬមិនមានសិទ្ធិ
- មិនឃើញប៊ូតុង action: ខ្វះ role/scope ឬមិនដល់ជំហាន

ពិនិត្យលម្អិតនៅ៖ `docs/correspondence-help-km/06-troubleshooting-faq.md`

## ៩) ស្តង់ដារគុណភាពទិន្នន័យ
