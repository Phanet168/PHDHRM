# SOP គ្រប់គ្រងថ្នាក់ និងឋានន្តរស័ក្តិ (HR)

ឯកសារនេះប្រើសម្រាប់ក្រុមការងារ HR/Admin និងអ្នកអនុម័ត ដើម្បីអនុវត្តដំណើរការតាមស្តង់ដារ មិនឱ្យជាន់គ្នា។

## 1) គោលបំណង

1. កំណត់លំហូរការងារតែមួយពីបង្កើតសំណើ រហូតដល់អនុម័តចុងក្រោយ។
2. បែងចែកតួនាទីឱ្យច្បាស់ថា អ្នកណាធ្វើអ្វី នៅដំណាក់កាលណា។
3. ធានាថាទិន្នន័យថ្នាក់/ឋានន្តរស័ក្តិ និងប្រវត្តិការងារត្រូវបានកត់ត្រាត្រឹមត្រូវ។

## 2) វិសាលភាព

SOP នេះគ្របដណ្តប់លើម៉ូឌុល៖ `គ្រប់គ្រងថ្នាក់ និងឋានន្តរស័ក្តិ`

ប្រភេទសំណើដែលគាំទ្រ៖

1. ដំឡើងថ្នាក់ប្រចាំឆ្នាំ (`annual_grade`)
2. ដំឡើងឋានន្តរស័ក្តិប្រចាំឆ្នាំ (`annual_rank`)
3. ដំឡើងតាមសញ្ញាបត្រ (`degree_based`)
4. ដំឡើងថ្នាក់កិត្តិយសមុននិវត្តន៍ (`honorary_pre_retirement`)

## 3) តួនាទី និងសិទ្ធិ

| តួនាទី | ភារកិច្ចសំខាន់ |
|---|---|
| HR Admin | កំណត់ឆ្នាំ, ពិនិត្យបញ្ជីគ្រប់លក្ខខណ្ឌ, បង្កើតសំណើ, តាមដានស្ថានភាព |
| អ្នកពិនិត្យ (ប្រធានការិយាល័យ/អនុប្រធាន) | ពិនិត្យ និងផ្តល់យោបល់លើសំណើ |
| អ្នកអនុម័តចុងក្រោយ (ប្រធានមន្ទីរ/អ្នកទទួលផ្ទេសិទ្ធ) | អនុម័ត ឬបដិសេធសំណើ |
| Super Admin | កំណត់ policy/permission និងត្រួតពិនិត្យប្រព័ន្ធ |

## 4) ស្ថានភាពសំណើ (Status Flow)

`proposed -> recommended -> approved`

ឬ

`proposed/recommended -> rejected`

ច្បាប់សំខាន់៖

1. មន្ត្រីម្នាក់មានបានតែ 1 pending request ក្នុងវដ្តដូចគ្នា។
2. `recommend` អនុវត្តបានតែពី `proposed`។
3. `approve` អនុវត្តបានតែពី `recommended`។
4. `reject` អាចធ្វើបានពី `proposed` ឬ `recommended`។

## 5) ដំណាក់កាលអនុវត្ត (មិនឱ្យជាន់គ្នា)

| ដំណាក់កាល | អ្នកអនុវត្ត | Tab/UI | លទ្ធផល |
|---|---|---|---|
| 1. កំណត់វដ្តប្រចាំឆ្នាំ | HR Admin | Dashboard | កំណត់ឆ្នាំ និង cutoff ថ្ងៃទី `01/04/YYYY` |
| 2. ពិនិត្យបញ្ជីគ្រប់លក្ខខណ្ឌ | HR Admin | បញ្ជីគ្រប់លក្ខខណ្ឌ | បញ្ជីអ្នកគ្រប់លក្ខខណ្ឌតាម rule |
| 3. បង្កើតសំណើ | HR Admin | បង្កើតសំណើ | Submit សំណើ និងក្លាយជា `proposed` |
| 4. ផ្តល់យោបល់ | អ្នកពិនិត្យ | ការអនុម័ត | ផ្លាស់ពី `proposed` ទៅ `recommended` |
| 5. អនុម័តចុងក្រោយ | អ្នកអនុម័តចុងក្រោយ | ការអនុម័ត | ក្លាយជា `approved` ឬ `rejected` |
| 6. កត់ត្រាប្រវត្តិ | ប្រព័ន្ធ + HR Admin | ប្រវត្តិដំឡើង | Update ថ្នាក់ថ្មី, បិទថ្នាក់ចាស់, កត់ Work History |
| 7. របាយការណ៍ | HR Admin | របាយការណ៍ | Print/Excel/PDF និងតាមដានអ្នកនៅសល់ |

## 6) Business Rules តាមស្តង់ដារ

1. Cutoff គណនា annual cycle ត្រឹមថ្ងៃទី `01 មេសា` រៀងរាល់ឆ្នាំ។
2. គ្រប់ 2 ឆ្នាំ = 730 ថ្ងៃ សម្រាប់លក្ខខណ្ឌ annual។
3. លើស 3 ឆ្នាំ = 1095 ថ្ងៃ ត្រូវចូល alert។
4. មុននិវត្តន៍ 1 ឆ្នាំ = 365 ថ្ងៃ សម្រាប់ honorary pre-retirement។
5. ប្រភេទ `degree_based` ត្រូវ review ដោយដៃ និងតម្រូវឯកសារគាំទ្រ។
6. សំណើអនុម័តរួច ត្រូវ lock មិនឱ្យកែផ្ទាល់។

## 7) SOP ប្រើប្រាស់តាម UI (Quick Guide)

1. ចូល `ផ្ទាំងសង្ខេប` -> ជ្រើសឆ្នាំ។
2. ចូល `បញ្ជីគ្រប់លក្ខខណ្ឌ` -> ពិនិត្យឈ្មោះគ្រប់លក្ខខណ្ឌ។
3. ចូល `បង្កើតសំណើ` -> បន្ថែមឈ្មោះ -> ចុច `ដាក់សំណើទាំងអស់`។
4. ចូល `ការអនុម័ត` -> ជ្រើសឈ្មោះ -> Recommend/Approve/Reject។
5. ចូល `ប្រវត្តិដំឡើង` -> ផ្ទៀងផ្ទាត់ថ្នាក់ថ្មីបានកត់ត្រា។
6. ចូល `របាយការណ៍` -> បោះពុម្ព ឬនាំចេញឯកសារ។

## 8) ករណីមិនគួរធ្វើ

1. មិនគួរបង្កើតសំណើស្ទួនសម្រាប់មន្ត្រីតែម្នាក់ក្នុងឆ្នាំដូចគ្នា។
2. មិនគួរអនុម័តដោយរំលង `recommended`។
3. មិនគួរផ្លាស់ថ្នាក់ដោយដៃក្រៅ flow អនុម័ត។

## 9) Checklist មុនបិទវដ្តប្រចាំឆ្នាំ

1. សំណើ pending ទាំងអស់ត្រូវបានសម្រេច (approved/rejected)។
2. មន្ត្រី approved ទាំងអស់មានថ្នាក់ថ្មីក្នុងប្រព័ន្ធ។
3. Work History កត់ត្រាគ្រប់ករណី។
4. របាយការណ៍បោះពុម្ព/Excel/PDF ត្រូវគ្នានឹងទិន្នន័យពិត។
